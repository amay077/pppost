# Threads 投稿機能の OpenSpec Proposal 作成

## Context

pppost は現在 Mastodon と Bluesky への同時投稿に対応している（Twitter は PPP-006 で廃除予定）。
本変更は、新たに **Threads** を投稿対象 SNS として追加するための OpenSpec proposal を作成する。

ユーザー合意済みのスコープ:
- **MVP = テキスト投稿のみ**（投稿対象チェックボックス・接続UI・投稿処理のみ）
- 認証/API は **Threads Graph API + Meta OAuth 2.0**、投稿はバックエンド Netlify Function 経由（Mastodon と同構成）
- **対象外（後続 proposal）**: 画像投稿、自投稿取得、リプライ選択、タイムライン取得、専用文字数カウンタ、トークン自動リフレッシュ

このフェーズで作るのは proposal ドキュメント（`.md`）のみ。**コード実装は行わない**（OpenSpec の Phase 1）。
実装は別途「実装して」の明示指示後（Phase 2 / openspec-apply）に行う。

## change-id

`PPP-009-add-threads-posting`（既存の `PPP-NNN-verb-name` 形式に準拠。コミットログは PPP-008 まで存在 → 次は 009）

## 既存パターン（踏襲方針）

各 SNS は以下の同型レイヤを持つ。Threads も完全に同型で追加する:
1. 型定義 — [func.ts](frontend/src/lib/func.ts) の `SettingData*` union / `SettingDataType<T>`
2. 永続化 — `localStorage` キー `ppp_setting_<type>`（`savePostSetting`/`loadPostSetting`、型駆動で変更不要）
3. 接続 UI — `frontend/src/lib/<Sns>Connection.svelte`
4. ランタイム状態 — [MainContent.ts](frontend/src/lib/MainContent.ts) の `postSettings` / `postTo`
5. 投稿関数 + switch 分岐 — `postToSns` 内の `postTo<Sns>()`
6. チェックボックス + アイコン — [MainContent.svelte](frontend/src/lib/MainContent.svelte)
7. バックエンド — `backend/netlify/functions/<sns>_token.js`（OAuth交換）, `<sns>_post.js`（投稿）

**Mastodon との決定的な違い**: Mastodon は OOB redirect でコードを手動コピペさせるが、
Threads は OOB 非対応。登録済み HTTPS `redirect_uri` へリダイレクトし、`MainContent.svelte` の
`onMount` で URL クエリの `code` を受信する方式が必須（旧 Twitter の onMount 受信パターンと同型）。

## Threads Graph API（確認済み仕様）

- 認可: `GET https://threads.net/oauth/authorize`（`scope=threads_basic,threads_content_publish`, `response_type=code`, `redirect_uri` 完全一致）
- 短命トークン: `POST https://graph.threads.net/oauth/access_token`（`grant_type=authorization_code` → `{access_token, user_id}`）
- 長命トークン(60日): `GET https://graph.threads.net/access_token`（`grant_type=th_exchange_token`）
- 投稿2段階: ①コンテナ作成 `POST https://graph.threads.net/v1.0/{user-id}/threads`（`media_type=TEXT`, `text`）→ `{id}` ②公開 `POST .../{user-id}/threads_publish`（`creation_id`）
- 本文上限: **500 文字**

## 作成する OpenSpec ファイル

`openspec/changes/PPP-009-add-threads-posting/` 配下:

### proposal.md
- **Why**: 投稿先 SNS の拡充。MVP はテキストのみで最小コストで価値提供
- **What Changes**: 型 `SettingDataThreads` 追加 / `ThreadsConnection.svelte` 新規 / `MainContent.ts` に `postToThreads`+switch / `MainContent.svelte` にチェックボックス・アイコン・onMount コールバック / `config.ts` に threads 設定 / backend `threads_token.js`・`threads_post.js` 新規 / 環境変数追加
- **Non-Goals**: 画像投稿・自投稿取得・リプライ選択・タイムライン取得・専用文字数カウンタ・トークン自動リフレッシュ
- **Impact**: Affected specs: `threads-posting`（新規） / Breaking changes: なし

### design.md（新パターン導入のため作成）
- **D1**: redirect 方式（同一タブ遷移 + onMount で `code` 受信）。Threads が OOB 非対応のため
- **D2**: 長命トークン交換はバックエンドで実施（`client_secret` をフロントに置かない）
- **D3**: テキスト単体は即時 publish。失敗時はエラー返却（MVP デフォルト）
- **D4**: 専用文字数カウンタは追加せず、500 文字超過はサーバ応答エラーとして扱う（MVP デフォルト）
- **D5**: `postToSns` の画像アップロード処理は他 SNS 用に維持し、Threads には画像を渡さない

### specs/threads-posting/spec.md（`## ADDED Requirements`）
- **Requirement 1**: Threads アカウント接続（OAuth）— 認可ページ遷移、長命トークン+user_id を `ppp_setting_threads` に保存、`client_secret` をフロントで扱わない (SHALL NOT)。Scenario: 接続する / 切断する
- **Requirement 2**: テキスト投稿（2段階）— コンテナ作成→公開、MVP では画像・リプライを送信しない (SHALL NOT)。Scenario: 投稿成功 / 投稿失敗（errors に `Threads`）
- **Requirement 3**: 文字数上限（500）— 超過時は投稿失敗として通知。Scenario: 500文字超過

### tasks.md（章立て、実装は Phase 2 で実行）
1. 型定義（func.ts: union/条件型に threads 追加）
2. config.ts（threads 設定 + env 読込）
3. ThreadsConnection.svelte（接続/切断UI、authorize 遷移）
4. MainContent.svelte（onMount コールバック、チェックボックス、アイコン、`postOfType`/`replyToPost` の `threads: undefined` 初期化）
5. MainContent.ts（postSettings/postTo/postToThreads/switch）
6. backend threads_token.js（短命→長命交換）
7. backend threads_post.js（2段階投稿）
8. 環境変数（.env.example 更新）
9. 動作検証（接続→投稿成功、切断、500文字超過、未接続時 disabled）

## 環境変数（proposal に記載、実装時に設定）

- backend: `PPPOST_THREADS_CLIENT_ID`, `PPPOST_THREADS_CLIENT_SECRET`, `PPPOST_THREADS_REDIRECT_URI`
- frontend: `VITE_THREADS_CLIENT_ID`, `VITE_THREADS_REDIRECT_URI`（backend の REDIRECT_URI と同値）
- localStorage キー: `ppp_setting_threads`

## 実装上の落とし穴（spec/tasks に反映）

1. `SettingType` 全網羅箇所（`MainContent.ts` の `postOfType` 初期化、`MainContent.svelte` の `replyToPost`/`postOfType` 初期化）に `threads: undefined` 追加が必要（TS 型エラー回避）
2. Threads は OOB 非対応 → Mastodon の手動コピペ UI を流用不可、リダイレクト + onMount 受信が必須
3. `redirect_uri` は authorize とトークン交換で完全一致（フロント/バック env を同値に）
4. 保存は長命トークン。`obtained_at` + `expires_in` で期限判定可能にしておく
