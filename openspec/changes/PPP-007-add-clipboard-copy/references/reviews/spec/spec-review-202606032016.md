# Review: PPP-007-add-clipboard-copy

## Summary

クリップボードコピー機能の proposal はフォーマット上 `openspec validate --strict` をパスし、構造も整っている。ただし spec のシナリオに具体的なフィードバック条件・失敗時の通知手段が定義されておらず、proposal の「コピー失敗（成功ではない）」のフィードバック経路が spec で網羅されていない点で Major 指摘が複数ある。

## Issues

### Issue 1: コピー「失敗」時のフィードバックが spec の要件・シナリオで網羅されていない
- **Severity**: Major
- **Location**: `specs/clipboard-copy/spec.md`
- **Line**: 9-24, 26-35
- **Description**: proposal.md L12 では「コピー成功・失敗をユーザーにフィードバックする」と明記し、tasks.md 2.4 / 3.2 でも「成功・失敗のフィードバック」「MIME 非対応時のエラーハンドリング」を求めている。しかし「テキストのクリップボードコピー」「画像のクリップボードコピー」の Requirement/Scenario には成功フィードバックしか定義がなく、API 対応環境であっても発生しうる失敗（権限拒否 `NotAllowedError`、`fetch` 失敗、`ClipboardItem` 書き込み失敗、画像 MIME 非対応）時のフィードバックが要件化されていない。「非対応環境のフォールバック」Requirement は API 自体が無い場合のみを扱っており、API はあるが書き込みに失敗するケースを覆っていない。proposal の意図（失敗フィードバック）が spec で網羅されていない。
- **Suggestion**: テキスト・画像各 Requirement に「コピーに失敗した場合、失敗をユーザーに通知しなければならない (SHALL)」を追記し、対応する失敗 Scenario（例: 画像 MIME がクリップボード非対応、書き込み権限拒否）を追加する。

### Issue 2: フィードバックの手段・条件が曖昧でテスト不可能
- **Severity**: Major
- **Location**: `specs/clipboard-copy/spec.md`
- **Line**: 18, 35, 51
- **Description**: 「コピー成功がユーザーにフィードバックされる」「別アプリへ画像として貼り付け可能である」「コピーに失敗した旨がユーザーに通知される」といった THEN が、具体的な観測可能条件を持たない。tasks.md 1.4 では「ボタン文言を一時的に変化させる」と具体化されているが、spec 側は手段・継続時間・復帰条件が未定義で、自動・手動を問わずテストの合否判定基準が定まらない。「フィードバックされる」「通知される」は実装に委ねられた曖昧表現。
- **Suggestion**: フィードバックの手段（例: ボタン文言を「コピー済み」に一定時間変更、トースト表示など）と観測条件を Scenario に明記する。proposal/tasks との整合を取り、少なくとも「成功時にボタン表示が `コピーしました` 等へ変化し、N 秒後に元へ戻る」のように検証可能な値・条件を入れる。

### Issue 3: 失敗時の画像コピーに対する検証タスクが欠落
- **Severity**: Major
- **Location**: `tasks.md`
- **Line**: 23-28
- **Description**: 「4. 動作検証」に成功系（4.1-4.3）と空テキスト無効化（4.4）の検証はあるが、tasks 3.1（非対応環境通知）・3.2（MIME 非対応エラーハンドリング）に対応する検証タスクが無い。spec の「非対応環境のフォールバック」Requirement / Scenario を検証するタスクが存在せず、Spec ↔ Tasks の検証網羅性を満たしていない。
- **Suggestion**: 「非対応ブラウザ（または `navigator.clipboard` を無効化した状態）でコピーボタン押下時に失敗通知が表示されることを確認」「クリップボード非対応 MIME 画像でエラー通知が出ることを確認」等の検証タスクを追加する。

### Issue 4: 画像コピー対象 URL の表現が proposal と spec で不一致
- **Severity**: Minor
- **Location**: `specs/clipboard-copy/spec.md` / `proposal.md`
- **Line**: spec 28、proposal 11
- **Description**: proposal は `croppedUrl ?? originalUrl`（表示中 URL）と明示しているが、spec の Requirement 本文は「クロップ済みの画像はクロップ後の画像（`croppedUrl`）を優先」とのみ記載し、クロップ未実施時に `originalUrl` をコピーする旨が要件文に明示されていない（Scenario でも未クロップ時の THEN が無い）。実コード（ImagePreview.svelte L158）の表示ロジックとも一致させるべき。
- **Suggestion**: Requirement に「クロップ未実施の場合は `originalUrl` をコピーする」を追記し、未クロップ画像コピーの Scenario を1つ追加する。

### Issue 5: テキスト Blob/巨大データ・複数画像同時コピー等のエッジケース未記載
- **Severity**: Minor
- **Location**: `specs/clipboard-copy/spec.md`
- **Line**: 26-41
- **Description**: 画像は1枚ずつコピーする前提（各プレビューにボタン）だが、spec には「コピー対象は押下したボタンに対応する1枚」である旨の明示がない。複数選択時にどの画像がコピーされるかは Scenario「GIVEN 1枚以上選択」だけでは曖昧。
- **Suggestion**: 「押下したボタンに対応する画像のみがコピーされる」ことを明示し、複数画像時の Scenario を補強する。
