# clipboard-copy Specification

## Purpose
入力テキストと選択画像を、ユーザー操作でクリップボードへコピーする機能。各 SNS アプリへの手動貼り付けを容易にする。
## Requirements
### Requirement: テキストのクリップボードコピー（Copy text to clipboard）

システムは、ユーザーがコピーボタンを押下したとき、入力テキストをクリップボードへコピーしなければならない (SHALL)。コピーボタンはアイコン表示とし、テキストが空の場合、コピーボタンは無効でなければならない (SHALL)。

コピー成功時、システムはテキストコピーボタンの表示を成功状態（チェックアイコン、ツールチップ `コピーしました`）へ変更し、2 秒後に元の表示へ戻さなければならない (SHALL)。

コピーに失敗した場合（`navigator.clipboard.writeText` が例外を送出、または権限拒否 `NotAllowedError` が発生した場合を含む）、システムはテキストコピーボタンの表示を失敗状態（✕アイコン、ツールチップ `コピー失敗`）へ変更し、2 秒後に元の表示へ戻さなければならない (SHALL)。システムは失敗を無言で握りつぶしてはならない (SHALL NOT)。

#### Scenario: テキストをコピーする（Copy entered text）

- **GIVEN** ユーザーが投稿テキストを入力している
- **WHEN** テキストコピーボタンを押下する
- **THEN** 入力テキストがクリップボードへコピーされる
- **AND** テキストコピーボタンの表示が成功状態（チェックアイコン、ツールチップ `コピーしました`）へ変化する
- **AND** 2 秒後にボタン表示が元へ戻る

#### Scenario: テキストコピーに失敗する（Copy text fails）

- **GIVEN** ユーザーが投稿テキストを入力している
- **AND** クリップボードへの書き込みが権限拒否（`NotAllowedError`）等で失敗する状態である
- **WHEN** テキストコピーボタンを押下する
- **THEN** テキストコピーボタンの表示が失敗状態（✕アイコン、ツールチップ `コピー失敗`）へ変化する
- **AND** 2 秒後にボタン表示が元へ戻る

#### Scenario: テキストが空のときは無効（Disabled when empty）

- **GIVEN** 投稿テキストが空である
- **WHEN** テキストコピーボタンの状態を確認する
- **THEN** ボタンは無効化されている

### Requirement: 画像のクリップボードコピー（Copy image to clipboard）

システムは、ユーザーが画像のコピーボタンを押下したとき、その画像データをクリップボードへコピーしなければならない (SHALL)。各画像プレビューはそれぞれコピーボタンを持ち、コピー対象は押下したボタンに対応する 1 枚のみでなければならない (SHALL)。複数画像が選択されていても、押下されていない画像はコピーしてはならない (SHALL NOT)。

コピー対象の画像 URL は、クロップ済みの場合はクロップ後の画像（`croppedUrl`）を、クロップ未実施の場合は元画像（`originalUrl`）をコピーしなければならない (SHALL)（すなわち表示中の URL `croppedUrl ?? originalUrl` をコピーする）。

コピー成功時、システムは押下した画像コピーボタンの表示を成功状態（例: アイコンまたはツールチップを `コピーしました` へ変更）へ変更し、2 秒後に元の表示へ戻さなければならない (SHALL)。

コピーに失敗した場合（画像 MIME タイプがクリップボード非対応、`fetch` による Blob 取得失敗、`ClipboardItem` への書き込み失敗、権限拒否 `NotAllowedError` を含む）、システムは押下した画像コピーボタンの表示を失敗状態（例: アイコンまたはツールチップを `コピー失敗` へ変更）へ変更し、2 秒後に元の表示へ戻さなければならない (SHALL)。システムは失敗を無言で握りつぶしてはならない (SHALL NOT)。

#### Scenario: 画像をコピーする（Copy a selected image）

- **GIVEN** ユーザーが画像を 1 枚以上選択している
- **WHEN** 対象画像のコピーボタンを押下する
- **THEN** 押下したボタンに対応する 1 枚の画像データのみがクリップボードへコピーされる
- **AND** 別アプリへ画像として貼り付け可能である
- **AND** 押下した画像コピーボタンの表示が成功状態（`コピーしました`）へ変化し、2 秒後に元へ戻る

#### Scenario: 複数選択時に押下した 1 枚のみコピーする（Copy only the targeted image）

- **GIVEN** ユーザーが画像を 2 枚以上選択している
- **WHEN** 2 枚目の画像のコピーボタンを押下する
- **THEN** 2 枚目の画像データのみがクリップボードへコピーされる
- **AND** 他の画像はコピーされない

#### Scenario: 未クロップ画像のコピー（Original image is copied when not cropped）

- **GIVEN** ユーザーが画像を選択しており、その画像をクロップしていない
- **WHEN** その画像のコピーボタンを押下する
- **THEN** 元画像（`originalUrl`）がクリップボードへコピーされる

#### Scenario: クロップ済み画像のコピー（Cropped image is preferred）

- **GIVEN** ユーザーが画像をクロップ済みである
- **WHEN** その画像のコピーボタンを押下する
- **THEN** クロップ後の画像（`croppedUrl`）がクリップボードへコピーされる

#### Scenario: 画像 MIME 非対応でコピーに失敗する（Copy fails for unsupported MIME）

- **GIVEN** ユーザーが画像を選択している
- **AND** その画像の MIME タイプがクリップボードに非対応である、またはクリップボードへの書き込みが失敗する状態である
- **WHEN** その画像のコピーボタンを押下する
- **THEN** 押下した画像コピーボタンの表示が失敗状態（`コピー失敗`）へ変化する
- **AND** 2 秒後にボタン表示が元へ戻る

### Requirement: 非対応環境のフォールバック（Fallback for unsupported environments）

システムは、Clipboard API が利用できない環境ではコピー失敗をユーザーに通知しなければならない (SHALL)。失敗を無言で握りつぶしてはならない (SHALL NOT)。

#### Scenario: Clipboard API 非対応（Clipboard API unavailable）

- **GIVEN** ブラウザが `navigator.clipboard` または `ClipboardItem` に対応していない
- **WHEN** ユーザーがコピーボタンを押下する
- **THEN** 押下したコピーボタンの表示が失敗状態（`コピー失敗`）へ変化する
- **AND** 2 秒後にボタン表示が元へ戻る
